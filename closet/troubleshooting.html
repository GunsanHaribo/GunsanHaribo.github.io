1) 사건 요약

배치 실행 이후 메모리가 우상향하며 Full GC 이후에도 Old 영역 사용량이 감소하지 않는 현상 관찰. 간헐적으로 S3 업로드 실패(UnknownHostException) 및 DNS 관련 에러가 동반됨. ECS Fargate 환경.

2) 배경/환경

  • JDK: Amazon Corretto 21
  • Framework: Spring Boot 3.x (Web, Batch, JPA)
  • Infra: AWS ECS(Fargate), ALB, CloudFront, S3, RDS(PostgreSQL)
  • 옵션: -Xlog:gc*,safepoint,class+load,class+unload,metaspace=info
  • 관측 도구: JFR, GC 로그, Heap Dump, jstat

3) 증상

  • 배치 주기마다 Old Gen 사용량 상승, Full GC 후에도 해제 불충분
  • 간헐적 S3 업로드 실패: SdkClientException → UnknownHostException
  • 메타스페이스 경계 근접 시 응답 지연 증가 의심
[error] SdkClientException: UnknownHostException ... s3.ap-northeast-2.amazonaws.com
[gc] GC(42) Pause Full (System.gc()) 410ms
[info] jstat -gcutil: OU 72% → 75% (after Full GC)

5) 원인 분석

가설 A. 배치 작업 중 외부 리소스 핸들 누수(스트림/클라이언트) → Old Gen 축적.
가설 B. DNS 해석 실패로 재시도 폭증 → 커넥션 풀/버퍼 증가.
가설 C. LaunchedClassLoader 경로에 남은 참조(스레드 로컬/스케줄러)로 클래스 언로드 지연 → 메타스페이스 증가. 

// 예: S3Client 재사용 없이 매 요청마다 생성 (비권장)
try (S3Client s3 = S3Client.builder().build()) {
  s3.putObject(...); // 실패 시 재시도 누적
}

결론(현재 베스트 추정): 네트워크/DNS 불안정으로 인한 재시도 누적 + 배치 리소스 해제 지연이 결합되어 Old Gen 잔존 및 메타스페이스 압박을 유발.

6) 해결 / 우회

  • S3 클라이언트 라이프사이클 정리: 빈으로 싱글톤 관리, 커넥션 풀/타임아웃 명시
  • DNS 캐시/리졸버 점검: 컨테이너 /etc/resolv.conf 확인, 재시도/백오프 조정
  • 배치 리소스 명시 해제: 스트림/HttpClient/ThreadLocal 정리, 스케줄러 종료 훅 추가
  • 관측 강화: JFR(1h) + GC 로그 + 주기적 Heap Dump(이벤트 기반)
// S3Client Bean (예시)
@Configuration
public class S3Config {
  @Bean
  S3Client s3Client() {
    return S3Client.builder()
      .region(Region.AP_NORTHEAST_2)
      .retryPolicy(b -> b.numRetries(2))
      .build();
  }
}
# JVM flags (예시)
-Xlog:gc*,safepoint,class+load=info,class+unload=info,metaspace=info:stdout:time,uptime,level,tags
-XX:+HeapDumpOnOutOfMemoryError
-XX:HeapDumpPath=/dumps

7) 검증

  • JFR 타임라인에서 Class Loading/Buffer Pools 안정 추세 확인
  • Full GC 이후 Old Gen 사용량 하향 회귀 확인 (jstat)
  • S3 업로드 성공률 및 재시도 지표 정상화
$ jstat -gcutil <pid> 5s 12
 S0  S1  E   O   M   CCS  YGC  YGCT  FGC  FGCT  GCT
 0.0 0.0 12.3 52.1 85.0 78.0  41  1.234   2   0.410  1.644

8) 후속 액션

  • 배치 잡별 리소스 린팅 체크리스트 도입 (파일/네트워크/스레드)
  • 네트워크 오류 타입별 재시도 정책 분리 (DNS vs 5xx vs 타임아웃)
  • 주간 JFR 스냅샷 + GC 로그 보관/리뷰 루틴화
